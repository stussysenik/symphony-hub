# Feature Request

## Recommended form fields

- `Title` (property field, required)
- `What outcome do you want?` (long text, required)
- `Who is this for?` (text, optional)
- `Surface area` (dropdown: frontend, backend, mobile, design, cross-product)
- `Why now?` (dropdown: customer need, product roadmap, ops pain, polish, experiment)
- `Mockups / references` (long text, optional)

## Recommended default properties

- Status: `Triage`
- Label: `feature`
- Team: preset if this template belongs to one team only

## Description body

```md
## Outcome
Describe the capability or user-facing result you want.

## User problem
What pain point or opportunity does this address?

## Constraints
Note any platform, technical, or design constraints.

## Acceptance notes
Describe what success looks like from a user perspective.

## References
Attach mockups, examples, or related issues if available.
```
